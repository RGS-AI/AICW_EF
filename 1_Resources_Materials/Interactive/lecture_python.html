01 · What is Python?

A language that just… works

Python is an interpreted, object-oriented, high-level language. Each of those words means something specific — and together they explain why Python dominates data science.

Analogy — a Swiss Army knife. Low-level languages (like C) are like a raw kitchen knife: fast, precise, but hard to use. Python is like a Swiss Army knife: not the absolute fastest, but it has a tool for everything, and you can use it straight away.

Interpreted

Python runs your code one line at a time, right now. You do not wait for a full translation step. Errors are found instantly.

High-level

You write print("hello"), not machine instructions. Python handles memory and hardware for you.

Object-oriented

Everything in Python is an object — a bundle of data and actions. Lists, strings, even numbers are objects with built-in methods.

Why does it dominate data science?

FeatureBenefit for data science
Readable syntaxCode reads almost like English — easy to share and maintain
Huge library ecosystemNumPy, Pandas, Scikit-learn, TensorFlow — all free
Interactive notebooksRun one cell, see the result, adjust, repeat
CommunityMillions of answered questions, tutorials, and examples
Key insight: Python trades raw speed for developer speed. For data science — where you spend most time thinking, not waiting — that trade is almost always worth it.

02 · Compilers vs Interpreters

Translate now or translate later?

Your computer only understands machine code (zeros and ones). A translator turns your Python text into machine code. There are two ways to do that translation.

Analogy — book vs. live interpreter. A compiler is like translating an entire book before anyone reads it — slow upfront, but the translated book is fast to read. An interpreter is like a live translator in a meeting — they translate each sentence as you speak it, immediately, but they do it every time.

Compiler (e.g. C, C++)

Reads the whole program first. Produces a standalone executable. Very fast to run. Errors caught before running.

Interpreter (Python)

Reads and runs one line at a time. No separate build step. Errors appear at the exact failing line, immediately.

Why does this matter for learning?

With an interpreter, you can open a Python shell, type one line, and see what happens. This tight feedback loop is one reason Python is excellent for learning and exploration.

⚡ See the difference — how many steps to "Hello, World!"?

Click each button to walk through the steps. Notice how many extra steps a compiled language needs.

Pitfall: Being interpreted does NOT mean Python is slow for data science. NumPy and Pandas run C code underneath — you get Python's ease with near-C speed for heavy number crunching.

04 · Variables

Giving names to values

A variable is a named box that holds a value. Give it a name, put a value in, and use the name anywhere you need the value.

Analogy — sticky labels on boxes. Imagine a warehouse of boxes. A variable is a sticky label you put on a box. The label "profit_q1" tells you what is inside without opening every box. You can swap the box's contents later — the label stays the same.

Creating a variable

$$ \text{variable\_name} = \text{value} $$

The equals sign = here means assignment (store this value), not equality in the math sense.

profit_q1 = 50000      # integer
profit_q2 = 75000
rate = 0.08            # float (decimal)
name = "Alice"         # string (text)
is_open = True         # boolean (True/False)

05 · Data Types

What kind of thing is this value?

A data type tells Python what kind of value a variable holds — and which operations make sense on it. You cannot divide a name by two.

Analogy — different shaped containers. A milk carton, a wallet, and a filing cabinet each hold different things. You would not pour juice into a wallet. Data types are Python's version of "different shapes for different things."

⚡ Type detector — click a value, see its type

Type conversion

Sometimes you need to change a value's type. Use the type name as a function:

x = "42"          # x is a string — you cannot do math on it
y = int(x)         # y is now the integer 42
z = float("3.14")  # z is 3.14, a float
s = str(100)       # s is "100", a string again
Pitfall: "42" + 8 crashes with a TypeError. Python will not mix types silently. Convert first: int("42") + 8 = 50.

07 · Input & Output

Talking to the user

Every useful program takes information in and sends results out. Python has two simple functions for this: input() and print().

Analogy — a drive-through window. input() is the speaker where you place your order. print() is the window where you pick up the result.

print() — sending output

print("Hello, world!")          # text
print(42)                        # number
print("Score:", 95, "/ 100")    # multiple values, separated by spaces
name = "Alice"
print(f"Hello, {name}!")        # f-string: embed a variable in text

input() — reading input

input() always returns a string. To do math, convert it first.

height = float(input("Enter your height in metres: "))
weight = float(input("Enter your weight in kg: "))
bmi = weight / (height ** 2)
print(f"Your BMI is {bmi:.2f}")   # :.2f → show 2 decimal places

08 · Strings & Slicing

Working with text

A string is a sequence of characters — letters, digits, spaces, symbols — all in a row. Python gives you powerful tools to chop, search, and reshape strings.

Analogy — a train of carriages. Each character is one carriage. You can pick carriage 0, or carriages 2 to 5, or every other carriage. That is exactly what string slicing does.

Indexing — picking one character

$$ s[i] \quad \text{character at position } i \text{ (counting from 0)} $$
s = "Python"
print(s[0])   # 'P'   (first character)
print(s[-1])  # 'n'   (last character — negatives count from the end)

Slicing — picking a range

$$ s[\text{start} : \text{stop} : \text{step}] $$

Gives characters from index start up to (but not including) stop, stepping by step.

name = "Edunet Foundation"
first = name[:6]       # "Edunet"  (stop=6, start defaults to 0)
last  = name[7:]       # "Foundation"  (start=7, stop defaults to end)
rev   = name[::-1]     # reverse the whole string

Useful string methods

.upper() .lower()

Change case. "python".upper() → "PYTHON".

.replace(a, b)

Swap every occurrence of a with b.

.split(sep)

Break into a list at each separator.

.strip()

Remove leading and trailing whitespace.

len(s)

Number of characters. len("hi") → 2.

f"..."

f-string: embed variables directly in text.

09 · Data Structures — Lists

An ordered, changeable collection

A list holds many values in a row, in order. Unlike a string, you can change a list after creating it — add items, remove items, reorder them.

Analogy — a shopping cart. Your cart has items in a specific order (the order you placed them). You can add a new item, remove one you changed your mind about, or check what is at position 2. That is a list.

Creating and indexing

fruits = ["apple", "banana", "cherry", "date", "fig", "grape"]
print(fruits[0])    # "apple"    (index 0 = first)
print(fruits[-1])   # "grape"    (index -1 = last)
print(fruits[2:5])  # ["cherry", "date", "fig"]  (slice)

10 · Data Structures — Tuples

An ordered collection you cannot change

A tuple is like a list, but immutable — once you create it, you cannot add, remove, or change items. Use tuples for data that should stay fixed.

Analogy — a printed receipt. A list is a whiteboard — you can erase and rewrite. A tuple is a printed receipt: the items are set. You can read them, but you cannot change them without reprinting.

Creating a tuple

coords = (10.5, 20.3)          # parentheses (optional, but clear)
days   = ("Mon", "Tue", "Wed", "Thu", "Fri")
single = (42,)                 # one-item tuple needs the trailing comma!

print(coords[0])   # 10.5  (same indexing as lists)
print(days[1:4])   # ('Tue', 'Wed', 'Thu')

When to use a tuple

Fixed data: GPS coordinates, RGB colours, days of the week, database records. Tuples are also slightly faster than lists for reading.

What you can still do

Index (t[0]), slice (t[1:3]), measure (len(t)), search ("Mon" in days). You just cannot change the contents.

⚡ List vs Tuple — spot the difference

Pitfall: Forgetting the trailing comma on a single-item tuple. (42) is just the number 42 in parentheses. (42,) is a one-item tuple.

11 · Data Structures — Sets

Unique items, no order

A set stores items with two guarantees: no duplicates and no fixed order. It is optimised for membership tests and set operations (union, intersection, difference).

Analogy — a bag of coloured marbles (no duplicates). You reach in and pull one out — but you cannot ask "give me the third marble" because there is no order. You CAN ask "is there a red marble in here?" and get an answer instantly.

Creating and using sets

ws1 = {"Alice", "Bob", "Charlie", "David"}
ws2 = {"Eve", "Bob", "Frank", "David"}

print(ws1 & ws2)   # intersection: {'Bob', 'David'}
print(ws1 | ws2)   # union: all unique names
print(ws1 - ws2)   # difference: in ws1, not in ws2
print("Alice" in ws1)  # True  — very fast check

12 · Data Structures — Dictionaries

Look things up by name, not position

A dictionary stores key-value pairs. Instead of asking "give me item at index 3", you ask "give me the item whose key is 'name'".

Analogy — a real dictionary. You look up "photosynthesis", not "word number 4,312". The word is the key; the definition is the value. Python dictionaries work the same way, but the "words" can be numbers, strings, or tuples — anything immutable.

Creating and reading

student = {
    "name": "Alice",
    "age": 20,
    "grades": [85, 90, 78],
    "major": "Computer Science"
}
print(student["name"])     # "Alice"
print(student["grades"])   # [85, 90, 78]

13 · Control Flow

Making decisions in code

if, elif, and else let your program choose a path based on a condition. Without them, every program runs the same steps every time — no intelligence.

Analogy — a traffic light. Green → go. Orange → slow. Red → stop. The traffic light checks the current state and picks a behaviour. Your if/elif/else block does the same thing.

The structure

if condition_1:
    # runs when condition_1 is True
elif condition_2:
    # runs when condition_1 is False AND condition_2 is True
else:
    # runs when every condition above is False

Worked example — age category

14 · Loops — for

Do something for each item

A for loop runs a block of code once for each item in a sequence — a list, a string, a range of numbers, or a dictionary's items. It is the engine of almost all data processing.

Analogy — a conveyor belt in a factory. Each item comes down the belt. The worker does the same job on each one. When the belt is empty, the loop ends. The worker never has to count — Python tracks it.

The structure

for item in collection:
    # do something with item

Common patterns

# Loop over a list
scores = [88, 72, 95, 61]
for s in scores:
    print(s)

# Loop with an index
for i, s in enumerate(scores):
    print(f"Student {i+1}: {s}")

# Loop over a range of numbers
for i in range(1, 6):     # 1, 2, 3, 4, 5
    print(i)

# Loop over dictionary items
grades = {"Alice": 90, "Bob": 75}
for name, g in grades.items():
    print(f"{name}: {g}")

⚡ Grade printer — step through the loop

Press Next to advance one loop iteration. Watch the variable change each time.

Worked example — grading each student

Given a dictionary of scores, print a letter grade for each student. A ≥ 90, B ≥ 80, C ≥ 70, D ≥ 60, F otherwise.

scores = {"Alice": 92, "Bob": 75, "Charlie": 88, "David": 55}

for name, score in scores.items():
    if score >= 90:
        grade = "A"
    elif score >= 80:
        grade = "B"
    elif score >= 70:
        grade = "C"
    elif score >= 60:
        grade = "D"
    else:
        grade = "F"
    print(f"{name}: {score} → {grade}")
Key insight: range(n) gives integers 0, 1, …, n-1. range(a, b) gives a, a+1, …, b-1. range(a, b, step) adds a step. These three forms cover almost every counting need.

15 · Loops — while, break, continue

Loop until a condition changes

A while loop keeps running as long as a condition is True. Use it when you do not know in advance how many times you need to loop.

Analogy — filling a glass. You keep pouring until the glass is full. You do not decide upfront "pour exactly 5 times" — you pour, check, pour, check. That is a while loop.

Structure and example

total = 0
numbers = []
while total <= 100:
    n = int(input("Enter a number: "))
    total += n
    numbers.append(n)
print("Numbers:", numbers)
print("Total:", total)

break and continue

break — exit immediately

Stop the loop right now, no matter what the condition says. Useful for "find the first match and stop".

for n in [2, 5, -1, 8]:
    if n < 0:
        break          # stop here
    print(n)           # prints 2, 5

continue — skip this turn

Skip the rest of this iteration and jump to the next one. Useful for "skip unwanted items".

for n in [2, 5, -1, 8]:
    if n < 0:
        continue       # skip negative
    print(n)           # prints 2, 5, 8

⚡ Running total — watch while decide to stop

Click "Add number" to push a random number onto the list. The loop condition checks after each addition.

Pitfall: An infinite loop runs forever and freezes your program. Always make sure the condition inside a while loop can eventually become False — or include a break.

16 · Functions

Write once, use many times

A function is a named block of code that does one job. You define it once and call it by name whenever you need it — with different inputs each time.

Analogy — a recipe card. The recipe for banana bread is written once. Any time you want banana bread, you follow the recipe — with whatever bananas you have on hand. A function is your recipe card; calling it is baking the bread.

Defining and calling

$$ \text{def } \texttt{function\_name(parameters)} \texttt{:} $$
def calculate_average(numbers):
    """Returns the average of a list of numbers."""
    total = sum(numbers)
    count = len(numbers)
    return total / count

scores = [85, 90, 78, 92]
avg = calculate_average(scores)
print(f"Average: {avg:.2f}")     # Average: 86.25

17 · File Handling

Reading and writing files

Real data lives in files — CSV spreadsheets, text logs, configuration files. Python gives you simple tools to read from and write to them.

Analogy — a filing cabinet. Opening a file is like pulling open a drawer. Reading is like scanning the pages. Writing is like adding a new sheet. Closing is like pushing the drawer shut so no one else gets confused. Always close the drawer when you are done.

The flow

Every file operation follows four steps: Open → Read or Write → Process → Close.

ModeMeaningFile exists?File missing?
"r"ReadOKError
"w"Write (overwrite)Clears itCreates it
"a"AppendAdds to endCreates it
"r+"Read and writeOKError

The safe way — with statement

The with statement closes the file automatically, even if an error happens. Always use it.

# Writing
with open("grades.txt", "w") as f:
    students = [("Alice", "A"), ("Bob", "B+"), ("Charlie", "A-")]
    for name, grade in students:
        f.write(f"{name},{grade}\n")

# Reading
with open("grades.txt", "r") as f:
    for line in f:
        name, grade = line.strip().split(",")
        print(f"{name} got {grade}")

⚡ File operation step-through

Reading a CSV — worked example

# products.csv has lines like: Notebook,45
with open("products.csv", "r") as f:
    total = 0
    for line in f:
        parts = line.strip().split(",")
        name  = parts[0]
        price = float(parts[1])
        total += price
    print(f"Total cost: ₹{total:.2f}")
Pitfall: Always call .strip() on each line before splitting. Every line from a file ends with \n (a newline character). Without strip(), your last value has an invisible \n attached — and float("45\n") crashes.

🎯 Check yourself — active recall

What is the difference between a list and a tuple? Try to answer before revealing.

A list ([...]) is mutable — you can add, remove, and change items after creation. A tuple ((...)) is immutable — once created, it cannot change. Use lists when the data changes; use tuples for fixed data like coordinates or days of the week.

🎯 Check yourself — active recall

Why do you always need to convert the result of input() before doing math?

input() always returns a string, even if the user typed a number. Python will not silently mix a string and a number. You must convert first: int(input(...)) or float(input(...)), otherwise you get a TypeError.
